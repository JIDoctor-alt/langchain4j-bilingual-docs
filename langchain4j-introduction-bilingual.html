Welcome 欢迎

The goal of LangChain4j is to simplify integrating LLMs into Java applications. Here's how.

EN

LangChain4j began development in early 2023 amid the ChatGPT hype. We noticed a lack of Java counterparts to the numerous Python and JavaScript LLM libraries and frameworks, and we had to fix that!

We actively monitor community developments, aiming to quickly incorporate new techniques and integrations, ensuring you stay up-to-date. The library is under active development. While some features are still being worked on, the core functionality is in place, allowing you to start building LLM-powered apps now!

中文

LangChain4j 于 2023 年初 ChatGPT 热潮中启动开发。我们注意到 Java 生态缺乏与众多 Python 和 JavaScript LLM 库/框架对应的解决方案，于是决定改变这一现状！

我们持续关注社区动态，快速融入新技术和集成，确保你始终与时俱进。该库处于活跃开发阶段，核心功能已就绪，现在就可以开始构建 LLM 驱动的应用！

03

Not a Port of LangChain (Python) 并非 LangChain (Python) 的移植

EN

Despite the name, LangChain4j is not a Java port of LangChain (Python) — it is built for Java, not ported to it. It is an idiomatic Java library designed from the ground up around Java conventions:

  • Type safety
  • POJOs, annotations, interfaces
  • Dependency injection
  • Fluent APIs
  • First-class integrations with Quarkus, Spring Boot, Helidon, and Micronaut

Its API, internals, and release cycle are independent of the Python LangChain project.

中文

尽管名称相似，LangChain4j 并非 LangChain (Python) 的 Java 移植版——它是为 Java 而建，而非移植到 Java。这是一个从零开始设计的、符合 Java 习惯的库：

  • 类型安全
  • POJO、注解、接口
  • 依赖注入
  • 流式 API
  • 与 Quarkus、Spring Boot、Helidon 和 Micronaut 的一流集成

其 API、内部实现和发布周期独立于 Python LangChain 项目。

Framework Integration — For easier integration, LangChain4j also includes integration with Quarkus, Spring Boot, Helidon and Micronaut. 为便于集成，LangChain4j 还提供了与 Quarkus、Spring Boot、Helidon 和 Micronaut 的集成支持。

Two Levels of Abstraction 两层抽象

Low Level · 底层

Low Level

At this level, you have the most freedom and access to all the low-level components such as ChatModel, UserMessage, AiMessage, EmbeddingStore, Embedding, etc. These are the "primitives" of your LLM-powered application. You have complete control over how to combine them, but you will need to write more glue code.

拥有最大自由度，可直接访问 ChatModel、UserMessage、EmbeddingStore 等底层组件，完全掌控组合方式，但需编写更多胶水代码。

High Level · 高层

High Level

At this level, you interact with LLMs using high-level APIs like AI Services, which hides all the complexity and boilerplate from you. You still have the flexibility to adjust and fine-tune the behavior, but it is done in a declarative manner.

使用 AI Services 等高层 API 与 LLM 交互，隐藏所有复杂性和样板代码，同时保留以声明式方式调整和微调行为的灵活性。

Library Structure 库结构

EN

Modular Design

LangChain4j features a modular design, comprising:

  • langchain4j-core — Core abstractions (ChatModel, EmbeddingStore) and their APIs
  • langchain4j — Main module with document loaders, chat memory, AI Services, and high-level features
  • langchain4j-{integration} — Wide array of modules for LLM providers and embedding stores. Can be used independently. For additional features, import the main langchain4j dependency.
中文

模块化设计

LangChain4j 采用模块化设计，包括：

  • langchain4j-core — 核心抽象（ChatModel、EmbeddingStore）及其 API
  • langchain4j — 主模块，包含文档加载器、聊天记忆、AI Services 等高层功能
  • langchain4j-{integration} — 大量集成模块，可独立使用，也可配合主模块获取更多功能
