Structured Outputs

让 LLM 输出可预测的 JSON 结构 —— 从非结构化文本中提取 POJO、枚举、多态类型，三种方式层层递进：JSON Schema 最可靠，Prompting + JSON Mode 次之，纯 Prompting 兜底。

01

概述 / Overview

EN

"Structured Outputs" refers to the general ability of the LLM to generate outputs in a structured format, typically JSON. These outputs can be easily mapped to Java objects. Note: this is distinct from OpenAI's "Structured Outputs" feature which applies to both response format and tools.

中文

"结构化输出"指 LLM 以结构化格式（通常是 JSON）生成输出的能力。这些输出可以轻松映射到 Java 对象。注意：这与 OpenAI 的"Structured Outputs"功能不同，后者同时适用于响应格式和工具。

示例目标：从非结构化文本中提取 Person 对象（name、age、height、married）。"John is 42 years old, 1.75m tall, unmarried." → Person[name=John, age=42, height=1.75, married=false]

三种方式 / Three Approaches

1
JSON Schema
最可靠方式。在请求中指定 JSON Schema，LLM 返回严格符合 schema 的 JSON。
2
Prompting + JSON Mode
在提示词中描述格式，并启用 JSON Mode 确保输出为合法 JSON。
3
Pure Prompting
仅通过提示词引导格式，AI Service 自动追加格式指令并解析。可靠性最低。

02

JSON Schema 与 ChatModel / JSON Schema with ChatModel

EN

In the low-level ChatModel API, use ResponseFormat and JsonSchema when creating a ChatRequest. The JSON schema is specified in a dedicated API attribute — no free-form instructions needed in the prompt. Use JsonObjectSchema.builder() to define the root element with properties, types, and required fields.

中文

在低级 ChatModel API 中，创建 ChatRequest 时使用 ResponseFormat 和 JsonSchema。JSON Schema 在 API 专有属性中指定，无需在提示词中添加自由格式指令。使用 JsonObjectSchema.builder() 定义根元素及其属性、类型和必填字段。

Java ChatModel + JSON Schema 
// 1. 定义 JSON Schema
ResponseFormat responseFormat = ResponseFormat.builder()
    .type(JSON)  // TEXT (default) 或 JSON
    .jsonSchema(JsonSchema.builder()
        .name("Person")  // OpenAI 要求指定 schema 名称
        .rootElement(JsonObjectSchema.builder()
            .addStringProperty("name")
            .addIntegerProperty("age")
            .addNumberProperty("height")
            .addBooleanProperty("married")
            .required("name", "age", "height", "married")
            .build())
        .build())
    .build();

// 2. 发送请求
ChatRequest chatRequest = ChatRequest.builder()
    .responseFormat(responseFormat)
    .messages(UserMessage.from(text))
    .build();

ChatResponse chatResponse = chatModel.chat(chatRequest);
String output = chatResponse.aiMessage().text();
// {"name":"John","age":42,"height":1.75,"married":false}

Person person = new ObjectMapper().readValue(output, Person.class);
必填字段：必须显式指定 required()，否则所有字段被视为可选。根元素通常为 JsonObjectSchema，但 Bedrock/Azure/Mistral/Ollama/OpenAI 也支持 JsonRawSchema 作为根元素。
03

JsonSchemaElement 子类型 / JsonSchemaElement Subtypes

JsonObjectSchema
对象类型，包含嵌套属性。通常作为 JsonSchema 的根元素。支持 properties()、addProperty()、add{Type}Property() 三种添加方式。
JsonStringSchema
String / char / Character 类型。可设置 description 帮助 LLM 理解。
JsonIntegerSchema
int / Integer / long / Long / BigInteger 类型。
JsonNumberSchema
float / Float / double / Double / BigDecimal 类型。
JsonBooleanSchema
boolean / Boolean 类型。
JsonEnumSchema
枚举类型。通过 enumValues() 指定可选值列表。
JsonArraySchema
数组和集合（List、Set）。通过 items() 指定元素类型。
JsonReferenceSchema
支持递归（如 Person 有 Set<Person> children）。仅 Azure OpenAI、Mistral、OpenAI 支持。
JsonAnyOfSchema
支持多态（如 Shape 可以是 Circle 或 Rectangle）。仅 OpenAI、Azure OpenAI、Gemini 支持。
JsonNullSchema
支持 nullable 类型。
JsonRawSchema
使用自定义完整 JSON Schema 字符串。Bedrock、Azure、Mistral、Ollama、OpenAI、Gemini 支持。
JsonAnyOfSchema — 多态示例
Java Polymorphism 
// Circle 和 Rectangle 统一为 Shape
JsonSchemaElement circleSchema = JsonObjectSchema.builder()
    .addNumberProperty("radius").build();

JsonSchemaElement rectSchema = JsonObjectSchema.builder()
    .addNumberProperty("width").addNumberProperty("height").build();

JsonSchemaElement shapeSchema = JsonAnyOfSchema.builder()
    .anyOf(circleSchema, rectSchema).build();

// LLM 输出: {"shapes":[{"radius":5},{"width":10,"height":20}]}
JsonReferenceSchema — 递归示例
Java Recursion 
// Person 递归包含 children: Set<Person>
String reference = "person";

JsonObjectSchema jsonObjectSchema = JsonObjectSchema.builder()
    .addStringProperty("name")
    .addProperty("children", JsonArraySchema.builder()
        .items(JsonReferenceSchema.builder().reference(reference).build())
        .build())
    .required("name", "children")
    .definitions(Map.of(reference, /* 递归定义 */ ...))
    .build();
描述（Description）：所有 JsonSchemaElement 子类型（除 JsonReferenceSchema）都支持 description 属性。当 LLM 输出不理想时，可添加描述提供更多指导和正确示例，如 "The name of the person, for example: John Doe"。
04

JSON Schema 与 AI Services / JSON Schema with AI Services

EN

AI Services makes structured output extraction effortless. Simply define an interface method returning a POJO, and enable JSON Schema on the ChatModel. LangChain4j auto-generates the ResponseFormat with JsonSchema from the return type. The schema name defaults to the return type's simple class name (e.g., "Person").

中文

AI Services 让结构化输出提取变得极其简单。只需定义返回 POJO 的接口方法，并在 ChatModel 上启用 JSON Schema。LangChain4j 自动根据返回类型生成 ResponseFormat 和 JsonSchema。Schema 名称默认为返回类型的简单类名（如 "Person"）。

Java AI Service + JSON Schema 
interface PersonExtractor {
    Person extractPersonFrom(String text);
}

// 启用 JSON Schema 功能（以 OpenAI 为例）
ChatModel chatModel = OpenAiChatModel.builder()
    .apiKey(System.getenv("OPENAI_API_KEY"))
    .modelName("gpt-4o-mini")
    .supportedCapabilities(RESPONSE_FORMAT_JSON_SCHEMA)  // 启用
    .strictJsonSchema(true)
    .build();

PersonExtractor extractor = AiServices.create(PersonExtractor.class, chatModel);
Person person = extractor.extractPersonFrom(text);
// Person[name=John, age=42, height=1.75, married=false]
三大条件：同时满足以下条件时自动启用 JSON Schema —— (1) AI Service 方法返回 POJO，(2) ChatModel 支持 JSON Schema，(3) JSON Schema 功能已显式启用。在 Quarkus/Spring Boot 中，ChatModel 和 AI Service 的 Bean 会自动创建。

05

必填/可选字段与描述 / Required, Optional & Description

EN

AI Services: by default, all fields are optional (LLMs tend to hallucinate and populate missing data). Use @JsonProperty(required = true) to make fields required. For tools, all fields are required by default. Use @Description on classes and fields to guide the LLM.

中文

AI Services：默认所有字段为可选（LLM 倾向于在信息不足时编造数据）。使用 @JsonProperty(required = true) 标记必填。工具中默认所有字段为必填。使用 @Description 在类和字段上指导 LLM。

Java @Description + @JsonProperty 
@Description("a person")
record Person(
    @Description("person's first and last name, for example: John Doe")
    @JsonProperty(required = true)
    String name,

    @Description("person's age, for example: 42")
    int age,

    @Description("person's height in meters, for example: 1.78")
    double height,

    @Description("is person married or not, for example: false")
    boolean married
) {}
注意：枚举值上的 @Description 无效，不会被包含在生成的 JSON Schema 中。可选的基本类型字段（int、boolean 等）在 LLM 未提供值时会被初始化为默认值（0、false 等）。
06

多态类型 / Polymorphic Types

EN

AI Service methods can return polymorphic types — a base type whose concrete subtype is decided by the LLM at runtime. Two flavors: sealed interfaces/classes (no annotations needed) and Jackson @JsonSubTypes. A discriminator property (default "type") tells the parser which subtype to instantiate.

中文

AI Service 方法可返回多态类型 —— 基类型，具体子类型由 LLM 在运行时决定。两种方式：sealed 接口/类（无需注解）和 Jackson @JsonSubTypes。鉴别器属性（默认 "type"）告诉解析器应实例化哪个子类型。

Java Sealed Interface 
// Sealed — 无需额外注解，自动发现子类型
sealed interface Animal permits Dog, Cat {}
record Dog(String name, String breed) implements Animal {}
record Cat(String name, boolean indoor) implements Animal {}

interface AnimalExtractor {
    Animal extractAnimalFrom(String text);
}

// "Rex is a Labrador" → {"value":{"type":"Dog","name":"Rex","breed":"Labrador"}}
// 注意：多态选择被包装在 value 属性下（因为多数提供商不支持根级 anyOf）
Java @JsonSubTypes 
// Jackson 方式 — 自定义 wire name
@JsonTypeInfo(use = Id.NAME, property = "kind")
@JsonSubTypes({
    @JsonSubTypes.Type(value = Square.class, name = "square"),
    @JsonSubTypes.Type(value = Circle.class, name = "circle")
})
interface Shape {}
class Square { double side; }
class Circle { double radius; }

// 鉴别器值解析顺序：@JsonSubTypes.Type.name → @JsonTypeName → Class.getSimpleName()
defaultImpl — 幻觉容错
Java defaultImpl 
@JsonTypeInfo(use = Id.NAME, defaultImpl = UnknownTool.class)
@JsonSubTypes({
    @JsonSubTypes.Type(value = Hammer.class, name = "hammer"),
    @JsonSubTypes.Type(value = Wrench.class, name = "wrench")
})
interface Tool {}

// LLM 输出 "saw"（未知鉴别器）→ 返回 UnknownTool 实例，不会崩溃
@JsonTypeInfo 属性支持的值
useId.NAME, Id.SIMPLE_NAME
includeAs.PROPERTY (默认), As.EXISTING_PROPERTY
property任意值；留空时默认 "@type"
defaultImpl任意具体子类 —— 鉴别器缺失或未知时使用
visibletrue 保留鉴别器字段在反序列化 Bean 上
递归多态：支持 sealed interface ExpressionNode { Literal, BinaryOp }，其中 BinaryOp 包含 ExpressionNode left, right。需要模型支持 $ref/$defs（Azure OpenAI、Mistral、OpenAI）。
07

限制与回退 / Limitations & Fallback

EN

ChatModel limitations: JSON Schema doesn't work in streaming mode for OpenAI. JsonReferenceSchema and JsonAnyOfSchema are only supported by Azure OpenAI, Mistral, and OpenAI.

AI Services limitations: No streaming mode. JSON Schema must be explicitly enabled. When the LLM doesn't support JSON Schema or it's not enabled, AI Services fall back to prompting — auto-generating format instructions appended to the UserMessage.

中文

ChatModel 限制：OpenAI 的 JSON Schema 暂不支持流式模式。JsonReferenceSchema 和 JsonAnyOfSchema 仅 Azure OpenAI、Mistral、OpenAI 支持。

AI Services 限制：不支持流式模式。JSON Schema 必须显式启用。当 LLM 不支持 JSON Schema 或未启用时，AI Services 回退到 prompting —— 自动生成格式指令追加到 UserMessage 末尾。

提示：纯 Prompting 方式相当不可靠。如果 LLM 和提供商支持 JSON Schema，强烈建议优先使用。
