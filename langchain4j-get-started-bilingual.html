02

Adding Dependencies 添加依赖

As an example, let's import the OpenAI dependency using Maven or Gradle.

EN

Maven (pom.xml)

Add the OpenAI integration dependency. If you wish to use the high-level AI Services API, also add the main langchain4j dependency.

中文

Maven (pom.xml)

添加 OpenAI 集成依赖。如果希望使用高层 AI Services API，还需添加主 langchain4j 依赖。

pom.xml 
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-open-ai</artifactId>
    <version>1.11.8</version>
</dependency>

<!-- Optional: for high-level AI Services API -->
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j</artifactId>
    <version>1.11.8</version>
</dependency>
EN

Gradle (build.gradle)

For Gradle users, use the implementation configuration.

中文

Gradle (build.gradle)

对于 Gradle 用户，使用 implementation 配置。

build.gradle 
implementation 'dev.langchain4j:langchain4j-open-ai:1.11.8'
implementation 'dev.langchain4j:langchain4j:1.11.8'
03

Bill of Materials (BOM) 物料清单 (BOM)

EN

Use the langchain4j-bom to manage all LangChain4j module versions consistently. It always contains the latest versions of all modules.

Please note that while the BOM version is 1.11.8, many modules still have version 1.11.8-beta19, so there might be some breaking changes for these modules in the future.

中文

使用 langchain4j-bom 统一管理所有 LangChain4j 模块版本。它始终包含所有模块的最新版本。

请注意，虽然 BOM 版本是 1.11.8，但许多模块仍为 1.11.8-beta19 版本，未来可能存在破坏性变更。

pom.xml — dependencyManagement 
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>dev.langchain4j</groupId>
            <artifactId>langchain4j-bom</artifactId>
            <version>1.11.8</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>
04

SNAPSHOT Dependencies 快照依赖

EN

Newest Features

If you'd like to test the newest features before their official release, you can use the most recent SNAPSHOT dependency. Add the Sonatype snapshot repository and use the 1.11.8-SNAPSHOT version.

中文

最新特性

如果希望在正式发布前测试最新功能，可以使用最新的 SNAPSHOT 依赖。添加 Sonatype 快照仓库并使用 1.11.8-SNAPSHOT 版本。

pom.xml — SNAPSHOT repository 
<repositories>
  <repository>
    <name>Central Portal Snapshots</name>
    <id>central-portal-snapshots</id>
    <url>https://central.sonatype.com/repository/maven-snapshots/</url>
    <releases>
      <enabled>false</enabled>
    </releases>
    <snapshots>
      <enabled>true</enabled>
    </snapshots>
  </repository>
</repositories>

<dependencies>
    <dependency>
        <groupId>dev.langchain4j</groupId>
        <artifactId>langchain4j</artifactId>
        <version>1.11.8-SNAPSHOT</version>
    </dependency>
</dependencies>

Configuring the API Key 配置 API 密钥

EN

Import your API Key

Import your OpenAI API key. It's recommended to store your API keys in environment variables to reduce the risk of exposing them publicly.

中文

导入 API 密钥

导入你的 OpenAI API 密钥。建议将 API 密钥存储在环境变量中，以降低公开发布的风险。

API Key 
String apiKey = System.getenv("OPENAI_API_KEY");
06

Demo Key 演示密钥

What if I don't have an API key? · 没有 API 密钥怎么办？

If you don't have your own OpenAI API key, don't worry. You can temporarily use a demo key, which we provide for free for demonstration purposes.

如果你没有自己的 OpenAI API 密钥，不用担心。你可以临时使用我们免费提供的演示密钥。

Be aware that when using the demo key, all requests go through our proxy, which injects the real key before forwarding to the OpenAI API. We do not collect or use your data. The demo key has a quota, is restricted to the gpt-4o-mini model, and should only be used for demonstration purposes.

请注意，使用演示密钥时，所有请求都经过我们的代理，代理会在转发前注入真实密钥。我们不会收集或使用你的数据。演示密钥有配额限制，仅限 gpt-4o-mini 模型，仅供演示用途。

Demo Key Configuration 
OpenAiChatModel model = OpenAiChatModel.builder()
    .baseUrl("http://langchain4j.dev/demo/openai/v1")
    .apiKey("demo")
    .modelName("gpt-4o-mini")
    .build();
