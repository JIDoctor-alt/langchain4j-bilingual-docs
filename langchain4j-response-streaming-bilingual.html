Response Streaming 响应流式传输

01

Overview 概述

LLMs generate text one token at a time. Streaming the response token-by-token significantly improves UX — users can start reading almost immediately instead of waiting for the entire text.

EN

Why Streaming?

LLMs generate text one token at a time, so many LLM providers offer a way to stream the response token-by-token instead of waiting for the entire text to be generated. This significantly improves the user experience, as the user does not need to wait an unknown amount of time and can start reading the response almost immediately.

For the ChatModel and LanguageModel interfaces, there are corresponding StreamingChatModel and StreamingLanguageModel interfaces. These have a similar API but can stream the responses. They accept an implementation of the StreamingChatResponseHandler interface as an argument.

中文

为什么要流式传输？

LLM 是逐 token 生成文本的，因此许多 LLM 提供商提供了逐 token 流式传输响应的方式，而不是等待整个文本生成完毕。这显著改善了用户体验——用户无需等待未知的时间，几乎可以立即开始阅读响应。

ChatModel 和 LanguageModel 接口分别有对应的 StreamingChatModel 和 StreamingLanguageModel 接口。它们的 API 类似，但可以流式传输响应，并接受 StreamingChatResponseHandler 接口的实现作为参数。

02

StreamingChatResponseHandler 流式聊天响应处理器

The core interface for handling streaming responses. Implement it to define actions for each streaming event.

EN

Interface Definition

By implementing StreamingChatResponseHandler, you can define actions for the following events:

  • Partial textual response — onPartialResponse(String) or onPartialResponse(PartialResponse, PartialResponseContext)
  • Partial thinking/reasoning — onPartialThinking(PartialThinking) or with context
  • Partial tool call — onPartialToolCall(PartialToolCall) or with context
  • Complete tool call — onCompleteToolCall(CompleteToolCall)
  • Complete response — onCompleteResponse(ChatResponse)
  • Error — onError(Throwable)
中文

接口定义

通过实现 StreamingChatResponseHandler，你可以为以下事件定义操作：

  • 部分文本响应 — onPartialResponse(String) 或带上下文版本
  • 部分思考/推理 — onPartialThinking(PartialThinking) 或带上下文版本
  • 部分工具调用 — onPartialToolCall(PartialToolCall) 或带上下文版本
  • 完整工具调用 — onCompleteToolCall(CompleteToolCall)
  • 完整响应 — onCompleteResponse(ChatResponse)
  • 错误 — onError(Throwable)
StreamingChatResponseHandler.java 
public interface StreamingChatResponseHandler {

    default void onPartialResponse(String partialResponse) {}
    default void onPartialResponse(PartialResponse partialResponse, PartialResponseContext context) {}

    default void onPartialThinking(PartialThinking partialThinking) {}
    default void onPartialThinking(PartialThinking partialThinking, PartialThinkingContext context) {}

    default void onPartialToolCall(PartialToolCall partialToolCall) {}
    default void onPartialToolCall(PartialToolCall partialToolCall, PartialToolCallContext context) {}

    default void onCompleteToolCall(CompleteToolCall completeToolCall) {}

    void onCompleteResponse(ChatResponse completeResponse);

    void onError(Throwable error);
}
03

Handler Events 处理器事件

Each event in the handler corresponds to a specific moment in the streaming lifecycle. All methods except onCompleteResponse and onError are default methods.

onPartialResponse

Invoked when the next partial textual response is generated. Partial response text can consist of a single or more tokens. You can send the token directly to the UI as soon as it becomes available.

下一个部分文本响应生成时调用。可将 token 直接发送到 UI。

onPartialThinking

Invoked when the next partial thinking/reasoning text is generated. Partial thinking text can consist of a single or more tokens.

下一个部分思考/推理文本生成时调用。

onPartialToolCall

Invoked when the next partial tool call is generated. Useful for showing real-time progress of tool invocations.

下一个部分工具调用生成时调用，可用于展示工具调用实时进度。

onCompleteToolCall

Invoked when the LLM has completed streaming for a single tool call. The CompleteToolCall contains the full tool call information.

LLM 完成单个工具调用的流式传输时调用，包含完整工具调用信息。

onCompleteResponse

Invoked when the LLM has completed generation. The ChatResponse object contains the complete response (AiMessage) as well as ChatResponseMetadata.

LLM 完成生成时调用，包含完整的 AiMessage 和元数据。

onError

Invoked when an error occurs during streaming. Handle exceptions gracefully here — logging, retry logic, or user notification.

流式传输发生错误时调用，用于日志记录、重试或用户通知。

04

Full Example 完整示例

Below is an example of how to implement streaming with StreamingChatModel using the full handler implementation.

EN

StreamingChatModel with Full Handler

Create an OpenAiStreamingChatModel and pass a StreamingChatResponseHandler implementation. Each callback receives the streaming data as it arrives.

中文

StreamingChatModel 与完整处理器

创建 OpenAiStreamingChatModel 并传入 StreamingChatResponseHandler 实现。每个回调在数据到达时接收流式数据。

StreamingChatModel Example 
StreamingChatModel model = OpenAiStreamingChatModel.builder()
    .apiKey(System.getenv("OPENAI_API_KEY"))
    .modelName(GPT_4_O_MINI)
    .build();

String userMessage = "Tell me a joke";

model.chat(userMessage, new StreamingChatResponseHandler() {

    @Override
    public void onPartialResponse(String partialResponse) {
        System.out.println("onPartialResponse: " + partialResponse);
    }

    @Override
    public void onPartialThinking(PartialThinking partialThinking) {
        System.out.println("onPartialThinking: " + partialThinking);
    }

    @Override
    public void onPartialToolCall(PartialToolCall partialToolCall) {
        System.out.println("onPartialToolCall: " + partialToolCall);
    }

    @Override
    public void onCompleteToolCall(CompleteToolCall completeToolCall) {
        System.out.println("onCompleteToolCall: " + completeToolCall);
    }

    @Override
    public void onCompleteResponse(ChatResponse completeResponse) {
        System.out.println("onCompleteResponse: " + completeResponse);
    }

    @Override
    public void onError(Throwable error) {
        error.printStackTrace();
    }
});
05

LambdaStreamingResponseHandler Lambda 流式响应处理器

A more compact way to stream the response using lambda expressions. The LambdaStreamingResponseHandler utility class provides static methods.

onPartialResponse

onPartialResponse

Define actions for partial responses with a single lambda. The onPartialResponse() static method accepts a lambda that defines what to do with the partial response.

使用单个 lambda 定义部分响应操作。onPartialResponse() 静态方法接受一个 lambda 表达式。

onPartialResponseAndError

onPartialResponseAndError

Define actions for both onPartialResponse() and onError() events. The onPartialResponseAndError() method accepts two lambdas for partial response handling and error handling.

同时定义 onPartialResponse() 和 onError() 两个事件的操作。接受两个 lambda 分别处理部分响应和错误。

EN

Simple Lambda

You just call the onPartialResponse() static method with a lambda expression. This is the most compact way to stream.

中文

简单 Lambda

只需调用 onPartialResponse() 静态方法并传入 lambda 表达式。这是最简洁的流式传输方式。

LambdaStreamingResponseHandler Examples 
// Simple: handle partial response only
import static dev.langchain4j.model.LambdaStreamingResponseHandler.onPartialResponse;

model.chat("Tell me a joke", onPartialResponse(System.out::print));

// With error handling
import static dev.langchain4j.model.LambdaStreamingResponseHandler.onPartialResponseAndError;

model.chat("Tell me a joke", onPartialResponseAndError(
    System.out::print,
    Throwable::printStackTrace
));
06

Streaming Cancellation 流式传输取消

If you wish to cancel the streaming, you can do so from the context-aware handler methods via StreamingHandle.cancel().

EN

How to Cancel

If you wish to cancel the streaming, you can do so from one of the following methods:

  • onPartialResponse(PartialResponse, PartialResponseContext)
  • onPartialThinking(PartialThinking, PartialThinkingContext)
  • onPartialToolCall(PartialToolCall, PartialToolCallContext)

The context object contains the StreamingHandle, which can be used to cancel the streaming. When StreamingHandle.cancel() is called, LangChain4j will close the connection and stop the streaming. Once cancelled, StreamingChatResponseHandler will not receive any further callbacks.

中文

如何取消

如果需要取消流式传输，可以通过以下带上下文的方法之一进行：

  • onPartialResponse(PartialResponse, PartialResponseContext)
  • onPartialThinking(PartialThinking, PartialThinkingContext)
  • onPartialToolCall(PartialToolCall, PartialToolCallContext)

上下文对象包含 StreamingHandle，可用于取消流式传输。调用 StreamingHandle.cancel() 后，LangChain4j 将关闭连接并停止流式传输。一旦取消，StreamingChatResponseHandler 不会再收到任何回调。

Streaming Cancellation Example 
model.chat(userMessage, new StreamingChatResponseHandler() {

    @Override
    public void onPartialResponse(PartialResponse partialResponse, PartialResponseContext context) {
        process(partialResponse);
        if (shouldCancel()) {
            context.streamingHandle().cancel();
        }
    }

    @Override
    public void onCompleteResponse(ChatResponse completeResponse) {
        System.out.println("onCompleteResponse: " + completeResponse);
    }

    @Override
    public void onError(Throwable error) {
        error.printStackTrace();
    }
});
IMPORTANT — When StreamingHandle.cancel() is called, LangChain4j will close the connection and stop the streaming. Once StreamingHandle.cancel() has been called, StreamingChatResponseHandler will not receive any further callbacks. 调用 StreamingHandle.cancel() 后，LangChain4j 将关闭连接并停止流式传输。一旦取消，StreamingChatResponseHandler 不会再收到任何回调。
