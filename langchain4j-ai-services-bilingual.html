AI Services

以声明式接口隐藏 LLM 交互的复杂性，如同 Spring Data JPA 之于数据库 —— 你只需定义接口，LangChain4j 负责实现。

01

概述 / Overview

EN

So far, we have been covering low-level components like ChatModel, ChatMessage, ChatMemory, etc. Working at this level is very flexible and gives you total freedom, but it also forces you to write a lot of boilerplate code. Since LLM-powered applications usually require multiple components working together and often involve multiple interactions, orchestrating them all becomes cumbersome.

中文

到目前为止，我们一直在讨论 ChatModel、ChatMessage、ChatMemory 等底层组件。在这个层面工作非常灵活，给予你完全的自由，但也迫使你编写大量样板代码。由于 LLM 驱动的应用程序通常需要多个组件协同工作，并涉及多次交互，编排它们会变得非常繁琐。

EN

We want you to focus on business logic, not on low-level implementation details. Thus, there are currently two high-level concepts: AI Services and Chains (legacy). AI Services are tailored for Java, hiding the complexities of interacting with LLMs and other components behind a simple API — very similar to Spring Data JPA or Retrofit.

中文

我们希望您专注于业务逻辑，而非底层实现细节。因此，目前有两个高级概念：AI Services 和 Chains（遗留）。AI Services 专为 Java 量身定制，将 LLM 和其他组件的交互复杂性隐藏在简洁的 API 背后 —— 与 Spring Data JPA 或 Retrofit 非常相似。

02

最简单的 AI Service / Simplest AI Service

EN

Define an interface with a single method, then create low-level components. Use AiServices.create() to get a proxy instance. In Quarkus and Spring Boot, autoconfiguration handles bean creation — you can simply inject/autowire the Assistant.

中文

定义一个带单个方法的接口，然后创建底层组件。使用 AiServices.create() 获取代理实例。在 Quarkus 和 Spring Boot 中，自动配置会处理 Bean 的创建 —— 你只需注入/自动装配即可。

Java Simplest Example 
interface Assistant {
    String chat(String userMessage);
}

// 创建底层 ChatModel
ChatModel model = OpenAiChatModel.builder()
    .apiKey(System.getenv("OPENAI_API_KEY"))
    .modelName(GPT_4_O_MINI)
    .build();

// 创建 AI Service 代理
Assistant assistant = AiServices.create(Assistant.class, model);

String answer = assistant.chat("Hello");
System.out.println(answer); // Hello, how can I help you?
工作原理：AiServices 创建代理对象，自动将 String 输入转换为 UserMessage，调用 ChatModel 后将 AiMessage 转换回 String 返回。目前使用反射实现，也在考虑其他替代方案。
03

注解驱动 / Annotation-Driven

EN

@SystemMessage — defines a system prompt template. Can load from resources: @SystemMessage(fromResource = "my-prompt-template.txt").

@UserMessage — defines a user prompt template with {{it}} or named variables via @V("name"). Can also load from resources.

中文

@SystemMessage — 定义系统提示模板。可从资源加载：@SystemMessage(fromResource = "my-prompt-template.txt")。

@UserMessage — 定义用户提示模板，使用 {{it}} 或通过 @V("name") 命名变量。也可从资源加载。

Java @SystemMessage 
interface Friend {

    @SystemMessage("You are a good friend of mine. Answer using slang.")
    String chat(String userMessage);
}

Friend friend = AiServices.create(Friend.class, model);
String answer = friend.chat("Hello"); // Hey! What's up?
Java @UserMessage + @V 
interface Friend {

    @UserMessage("You are a good friend of mine. Answer using slang. {{message}}")
    String chat(@V("message") String userMessage);
}

// 在 Quarkus / Spring Boot 中 @V 非必需（-parameters 已启用）
04

系统消息提供者与转换器 / System Message Provider & Transformer

EN

SystemMessageProvider — dynamically defines system messages based on a chat memory ID (user or conversation).

SystemMessageTransformer — dynamically modifies the system message on every invocation, after resolution but before the chatRequestTransformer runs. Useful for appending/prepending content.

中文

SystemMessageProvider — 基于 chat memory ID 动态定义系统消息。

SystemMessageTransformer — 在每次调用时动态修改系统消息，在解析之后、chatRequestTransformer 运行之前执行。适用于追加或前置内容。

Java Provider + Transformer 
Friend friend = AiServices.builder(Friend.class)
    .chatModel(model)
    .systemMessageProvider(chatMemoryId ->
        "You are a good friend of mine. Answer using slang.")
    .systemMessageTransformer((systemMessage, context) ->
        systemMessage + " Tenant: " +
        context.invocationParameters().get("tenant") + ".")
    .build();
05

按调用配置参数 / ChatRequestParameters

EN

Configure parameters (temperature, toolsChoice, max tokens, etc.) on a per-call basis. Some requests may need more creativity (higher temperature), others more determinism. The method accepts a ChatRequestParameters argument that is merged during invocation.

中文

按每次调用配置参数（temperature、toolsChoice、max tokens 等）。某些请求可能需要更多创造性（高 temperature），另一些需要更高的确定性。方法接受 ChatRequestParameters 参数，在调用时合并。

Java Per-Call Parameters 
interface AssistantWithChatParams {
    String chat(@UserMessage String userMessage,
                   ChatRequestParameters params);
}

ChatRequestParameters customParams = ChatRequestParameters.builder()
    .temperature(0.85)
    .build();

String answer = assistant.chat("Hi there!", customParams);

有效方法签名示例 / Valid Method Signatures

UserMessage only
String chat(String userMessage);
String chat(@UserMessage String userMessage);
String chat(@UserMessage String userMessage, @V("country") String country);
String chat(@UserMessage String userMessage, @UserMessage Content content);
String chat(@UserMessage String userMessage, @UserMessage List<Content> contents);
@UserMessage("What is the capital of {{it}}?") String chat(String country);
SystemMessage + UserMessage
@SystemMessage("Given a name of a country, answer with its capital") String chat(String userMessage);
@SystemMessage("Given a name of a country, {{answerInstructions}}") @UserMessage("{{country}}") String chat(@V("answerInstructions") String ins, @V("country") String country);
Multimodality
String chat(@UserMessage String userMessage, @UserMessage ImageContent image);
String chat(@UserMessage String userMessage, @UserMessage List<ImageContent> images);
String chat(@UserMessage AudioContent audio, @UserMessage ImageContent image);
Return Types：方法可返回 String、任何结构化输出类型，或包装在 Result<T> 中以获取额外元数据（TokenUsage、Sources、ToolExecutions、FinishReason 等）。
07

结构化输出 / Structured Outputs

EN

Change the return type from String to any other type to receive structured output. Supports boolean, enum, and custom POJOs. Use @Description on fields to help the LLM understand the schema better.

中文

将返回类型从 String 改为其他类型即可接收结构化输出。支持 boolean、enum 和自定义 POJO。在字段上使用 @Description 帮助 LLM 更好地理解 schema。

Java Boolean · Enum · POJO 
// boolean 返回
interface SentimentAnalyzer {
    @UserMessage("Does {{it}} has a positive sentiment?")
    boolean isPositive(String text);
}

// enum 返回
enum Priority { CRITICAL, HIGH, LOW }

interface PriorityAnalyzer {
    @UserMessage("Analyze the priority of: {{it}}")
    Priority analyzePriority(String issueDescription);
}

// POJO 返回
class Person {
    @Description("first name of a person")
    String firstName;
    String lastName;
    LocalDate birthDate;
    Address address;
}

interface PersonExtractor {
    @UserMessage("Extract person info from {{it}}")
    Person extractPersonFrom(String text);
}
08

JSON 模式 / JSON Mode

EN

When extracting custom POJOs, enable "JSON mode" to force the LLM to respond with valid JSON. JSON mode is for structured data extraction; tools/function calling is for when the LLM needs to perform actions. Each provider has its own configuration API.

中文

提取自定义 POJO 时，建议启用"JSON 模式"强制 LLM 返回有效 JSON。JSON 模式用于结构化数据提取；工具/函数调用用于 LLM 需要执行操作时。每个提供商有其独特的配置方式。

OpenAI (新模型: gpt-4o-mini)
.supportedCapabilities(RESPONSE_FORMAT_JSON_SCHEMA) .strictJsonSchema(true)
OpenAI (旧模型: gpt-3.5-turbo)
.responseFormat("json_object")
Azure OpenAI
.responseFormat( new ChatCompletionsJsonResponseFormat())
Vertex AI Gemini
.responseMimeType("application/json") // 或显式 schema: .responseSchema( SchemaHelper.fromClass(Person.class))
Google AI Gemini
.responseFormat(ResponseFormat.JSON) // 或显式 schema: .responseFormat(ResponseFormat.builder() .type(JSON) .jsonSchema(JsonSchemas .jsonSchemaFrom(Person.class).get()) .build())
Mistral AI
.supportedCapabilities(RESPONSE_FORMAT_JSON_SCHEMA) .strictJsonSchema(true)
Ollama
.responseFormat(JSON)
其他提供商：如果底层模型不支持 JSON 模式，可使用 prompt engineering 并降低 temperature 来提高确定性。
09

流式响应 / Streaming

EN

Use TokenStream return type to stream response token-by-token. Rich callback API: onPartialResponse, onPartialThinking, onPartialToolCall, beforeToolExecution, onToolExecuted, onCompleteResponse, onError. Also supports Flux<String> via the langchain4j-reactor module.

中文

使用 TokenStream 返回类型实现逐 token 流式响应。丰富的回调 API：onPartialResponse、onPartialThinking、onPartialToolCall、beforeToolExecution、onToolExecuted、onCompleteResponse、onError。也支持通过 langchain4j-reactor 模块使用 Flux<String>。

Java TokenStream 
interface Assistant {
    TokenStream chat(String message);
}

StreamingChatModel model = OpenAiStreamingChatModel.builder()
    .apiKey(System.getenv("OPENAI_API_KEY"))
    .modelName(GPT_4_O_MINI)
    .build();

Assistant assistant = AiServices.create(Assistant.class, model);

tokenStream
    .onPartialResponse(partial -> System.out.println(partial))
    .onPartialThinking(thinking -> System.out.println(thinking))
    .onPartialToolCall(toolCall -> System.out.println(toolCall))
    .beforeToolExecution(before -> System.out.println(before))
    .onToolExecuted(execution -> System.out.println(execution))
    .onCompleteResponse(futureResponse::complete)
    .onError(futureResponse::completeExceptionally)
    .start();
EN

Streaming Cancellation: Use onPartialResponseWithContext or onPartialThinkingWithContext to access StreamingHandle.cancel(). Once cancelled, no further callbacks are received.

中文

流式取消：使用 onPartialResponseWithContext 或 onPartialThinkingWithContext 访问 StreamingHandle.cancel()。一旦取消，将不再收到任何回调。

聊天记忆 / Chat Memory

EN

AI Service can use chat memory to "remember" previous interactions. For multi-user scenarios, use ChatMemoryProvider with @MemoryId annotation. Extend ChatMemoryAccess to access and evict per-conversation memories. Important: Do not call AI Service concurrently for the same @MemoryId.

中文

AI Service 可使用聊天记忆"记住"之前的交互。对于多用户场景，使用 ChatMemoryProvider 和 @MemoryId 注解。扩展 ChatMemoryAccess 可访问和清除各对话的记忆。重要：不要对相同的 @MemoryId 并发调用 AI Service。

Java ChatMemoryProvider + @MemoryId 
interface Assistant extends ChatMemoryAccess {
    String chat(@MemoryId int memoryId,
               @UserMessage String message);
}

Assistant assistant = AiServices.builder(Assistant.class)
    .chatModel(model)
    .chatMemoryProvider(memoryId ->
        MessageWindowChatMemory.withMaxMessages(10))
    .build();

String answerToKlaus = assistant.chat(1, "Hello, my name is Klaus");
String answerToFrancine = assistant.chat(2, "Hello, my name is Francine");

// 访问和清除记忆
List<ChatMessage> msgs = assistant.getChatMemory(1).messages();
boolean evicted = assistant.evictChatMemory(2);
11

工具调用 / Tools (Function Calling)

EN

Configure AI Service with tools that the LLM can use. Annotate methods with @Tool. LangChain4j automatically executes requested tools and feeds results back to the LLM for the final answer.

中文

为 AI Service 配置 LLM 可用的工具。使用 @Tool 注解方法。LangChain4j 自动执行请求的工具并将结果反馈给 LLM 以生成最终答案。

Java @Tool 
class Tools {

    @Tool
    int add(int a, int b) {
        return a + b;
    }

    @Tool
    int multiply(int a, int b) {
        return a * b;
    }
}

Assistant assistant = AiServices.builder(Assistant.class)
    .chatModel(model)
    .tools(new Tools())
    .build();

String answer = assistant.chat("What is 1+2 and 3*4?");
// LLM 请求执行 add(1,2) 和 multiply(3,4)，LangChain4j 自动执行

RAG 检索增强 / Retrieval-Augmented Generation

EN

Naive RAG: Configure with ContentRetriever.
Advanced RAG: Use RetrievalAugmentor for query transformation, re-ranking, content aggregation, etc.
RAG as a Tool: Encapsulate ContentRetriever in a @Tool — the LLM decides when to trigger retrieval, avoiding unnecessary searches for simple queries.

中文

朴素 RAG：配置 ContentRetriever。
高级 RAG：使用 RetrievalAugmentor 实现查询转换、重排序、内容聚合等。
RAG 作为工具：将 ContentRetriever 封装在 @Tool 中 —— LLM 自主决定何时触发检索，避免对简单查询进行不必要的搜索。

Java Naive RAG + Advanced RAG 
// 朴素 RAG
ContentRetriever contentRetriever = new EmbeddingStoreContentRetriever(
    embeddingStore, embeddingModel);

Assistant assistant = AiServices.builder(Assistant.class)
    .chatModel(model)
    .contentRetriever(contentRetriever)
    .build();

// 高级 RAG
RetrievalAugmentor retrievalAugmentor = DefaultRetrievalAugmentor.builder()
    .queryTransformer(...)
    .queryRouter(...)
    .contentAggregator(...)
    .contentInjector(...)
    .executor(...)
    .build();
场景 A — 一般对话
用户输入：Hello, how are you today?
LLM 直接从内部知识回答，不调用检索工具。
场景 B — 技术问题
用户输入：How do I configure a ContentRetriever?
LLM 识别技术意图，调用 search()，基于检索到的文档生成回答。
13

链式调用与自动审核 / Chaining & Auto-Moderation

EN

Auto-Moderation: Configure a ModerationModel — when inappropriate content is detected, a ModerationException is thrown.

Chaining: Break complex logic into smaller, specialized AI Services. Use deterministic and LLM-powered control flow (if/else with boolean, switch with enum, loops with numeric returns). Each AI Service can be mocked, integration-tested, and optimized independently.

中文

自动审核：配置 ModerationModel —— 检测到不当内容时抛出 ModerationException。

链式调用：将复杂逻辑拆分为更小、更专业的 AI Service。使用确定性和 LLM 驱动的控制流（boolean 实现 if/else、enum 实现 switch、数值类型实现循环）。每个 AI Service 可独立模拟、集成测试和优化。

Java Chaining Example 
interface GreetingExpert {
    @UserMessage("Is the following text a greeting? Text: {{it}}")
    boolean isGreeting(String text);
}

interface ChatBot {
    @SystemMessage("You are a polite chatbot of Miles of Smiles.")
    String reply(String userMessage);
}

class MilesOfSmiles {
    public String handle(String userMessage) {
        if (greetingExpert.isGreeting(userMessage)) {
            return "Greetings from Miles of Smiles!";
        } else {
            return chatBot.reply(userMessage);
        }
    }
}

// 简单任务用便宜的 Llama2，复杂任务用 GPT-4 + RAG
GreetingExpert greetingExpert = AiServices.create(GreetingExpert.class, llama2);
ChatBot chatBot = AiServices.builder(ChatBot.class)
    .chatModel(gpt4)
    .contentRetriever(milesOfSmilesContentRetriever)
    .build();
设计原则：更小、更专注的组件更易于开发、测试、维护和理解。不要让 LLM 一次性访问所有工具 —— 根据场景按需提供，既节省 token 成本，又降低幻觉风险。
