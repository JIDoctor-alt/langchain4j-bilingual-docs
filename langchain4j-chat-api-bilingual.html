01 / 概述

Overview
概述

English

LangChain4j offers two main abstraction levels for interacting with LLMs:

  • Low-level: ChatModel and LanguageModel — the core building blocks.
  • High-level: AiServices — a declarative API that handles message formatting, tool calls, and more automatically.

This tutorial focuses on the low-level APIs. For most use cases, AiServices is the recommended starting point.

中文

LangChain4j 提供两个主要的抽象级别来与 LLM 交互：

  • 低级 API：ChatModel 和 LanguageModel —— 核心构建块。
  • 高级 API：AiServices —— 声明式 API，自动处理消息格式化、工具调用等。

本教程专注于低级 API。对于大多数用例，推荐从 AiServices 开始。

02 / 接口区别

ChatModel vs LanguageModel
ChatModel 与 LanguageModel 对比

English

ChatModel

ChatModel is the most common API. It accepts a sequence of ChatMessages (with their types) as input and returns an AiMessage as output. Chat models are designed for conversational, multi-turn interactions.

This is the recommended API for most use cases because it preserves message type information, enabling features like system prompts, tool calls, and structured conversation history.

中文

ChatModel

ChatModel 是最常用的 API。它接受一系列 ChatMessage（及其类型）作为输入，返回一个 AiMessage 作为输出。聊天模型专为对话式多轮交互而设计。

这是大多数用例的推荐 API，因为它保留了消息类型信息，支持系统提示、工具调用和结构化的对话历史等功能。

English

LanguageModel

LanguageModel is a simpler, legacy API. It accepts a single String as input and returns a String as output. It does not support message types, system prompts, or tool calls directly.

Use this API only for simple text-in/text-out scenarios where conversation context is not needed.

中文

LanguageModel

LanguageModel 是一个更简单的遗留 API。它接受单个 String 作为输入，返回一个 String 作为输出。它不直接支持消息类型、系统提示或工具调用。

仅在不需要对话上下文的简单文本输入/输出场景中使用此 API。

03 / 消息类型

ChatMessage Types
ChatMessage 类型

LangChain4j supports five message types, each with a distinct role in the conversation.
UserMessage

User Message

The message sent by the end user — the "human" side of the conversation.

由最终用户发送的消息 —— 对话中的"人类"方。

AiMessage

AI Message

The response generated by the AI model. This is the only message type that ChatModel returns.

由 AI 模型生成的响应。这是 ChatModel 唯一返回的消息类型。

SystemMessage

System Message

Instructions that define the AI's behavior, persona, or constraints for the entire conversation.

定义 AI 在整个对话中的行为、角色或约束的指令。

ToolExecutionResultMessage

Tool Execution Result

The result of a tool execution, returned to the model so it can incorporate the data into its response.

工具执行的结果，返回给模型以便将数据纳入其响应中。

CustomMessage

Custom Message

A user-defined message type for scenarios not covered by the built-in types.

用户自定义的消息类型，用于内置类型未覆盖的场景。

04 / ChatModel API

ChatModel Interface
ChatModel 接口

English

The ChatModel interface provides several methods for generating responses:

  • generate(ChatMessage...) — single message input
  • generate(List<ChatMessage>) — list of messages
  • generate(List<ChatMessage>, List<ToolSpecification>) — with tool support
  • generate(List<ChatMessage>, ToolSpecification...) — varargs tool support

Each method returns a Response<AiMessage> containing the generated message and metadata (token usage, finish reason, etc.).

中文

ChatModel 接口提供多个生成响应的方法：

  • generate(ChatMessage...) —— 单条消息输入
  • generate(List<ChatMessage>) —— 消息列表
  • generate(List<ChatMessage>, List<ToolSpecification>) —— 支持工具
  • generate(List<ChatMessage>, ToolSpecification...) —— 可变参数工具支持

每个方法返回一个 Response<AiMessage>，包含生成的消息和元数据（token 使用量、结束原因等）。

ChatModel.java 
public interface ChatModel {

    Response<AiMessage> generate(ChatMessage... messages);

    Response<AiMessage> generate(List<ChatMessage> messages);

    Response<AiMessage> generate(
        List<ChatMessage> messages,
        List<ToolSpecification> toolSpecifications
    );

    Response<AiMessage> generate(
        List<ChatMessage> messages,
        ToolSpecification... toolSpecifications
    );

    // ... additional methods
}
Best Practice: Always use SystemMessage at the beginning of the conversation to set the AI's behavior. Follow with UserMessage(s) and AiMessage(s) to maintain conversation history. 始终在对话开头使用 SystemMessage 来设定 AI 的行为。接续使用 UserMessage 和 AiMessage 来维护对话历史。
05 / 流式传输

StreamingChatModel
流式聊天模型

English

StreamingChatModel extends ChatModel to support token-by-token streaming. Instead of waiting for the full response, you receive tokens as they are generated.

This is useful for real-time UIs, chatbots, and scenarios where responsiveness matters. The streaming API uses StreamingChatResponseHandler to handle individual tokens.

中文

StreamingChatModel 扩展了 ChatModel，支持逐 token 流式传输。您无需等待完整响应，即可在 token 生成时即时接收。

这对于实时 UI、聊天机器人以及需要响应速度的场景非常有用。流式 API 使用 StreamingChatResponseHandler 来处理单个 token。

StreamingChatModel.java 
public interface StreamingChatModel {

    void generate(
        List<ChatMessage> messages,
        StreamingChatResponseHandler handler
    );

    void generate(
        List<ChatMessage> messages,
        List<ToolSpecification> toolSpecifications,
        StreamingChatResponseHandler handler
    );

    void generate(
        List<ChatMessage> messages,
        ToolSpecification... toolSpecifications,
        StreamingChatResponseHandler handler
    );
}

07 / 多模态

Multimodality
多模态支持

English

LangChain4j supports multimodal models that can process both text and images. Use UserMessage with ImageContent or Content to include images in your prompts.

Supported content types include TextContent and ImageContent, which can be mixed within a single UserMessage.

中文

LangChain4j 支持能够处理文本和图像的多模态模型。使用 UserMessage 配合 ImageContent 或 Content 在提示中包含图像。

支持的内容类型包括 TextContent 和 ImageContent，可以在单个 UserMessage 中混合使用。

Multimodal Example 
UserMessage userMessage = UserMessage.from(
    TextContent.from("Describe this image:"),
    ImageContent.from("https://example.com/image.jpg")
);

Response<AiMessage> response = model.generate(userMessage);
08 / 示例

Usage Examples
使用示例

English

Simple ChatModel Usage

The most basic usage pattern: create a model instance, send a user message, and receive an AI response.

中文

简单的 ChatModel 使用

最基本的用法：创建模型实例，发送用户消息，接收 AI 响应。

BasicExample.java 
ChatModel model = OpenAiChatModel.builder()
    .apiKey("demo")
    .modelName("gpt-4o-mini")
    .build();

String answer = model.generate("Say 'Hello World'");
System.out.println(answer);
// Hello World
English

Multi-turn Conversation with System Prompt

A more realistic example showing a conversation with system instructions and multiple turns.

中文

带系统提示的多轮对话

一个更真实的示例，展示带有系统指令和多轮对话的交互。

MultiTurnExample.java 
ChatModel model = OpenAiChatModel.builder()
    .apiKey("demo")
    .modelName("gpt-4o-mini")
    .build();

SystemMessage systemMessage = SystemMessage.from(
    "You are a helpful assistant who speaks like a pirate."
);

UserMessage userMessage1 = UserMessage.from(
    "Tell me a joke about Java developers."
);

Response<AiMessage> response1 = model.generate(
    systemMessage, userMessage1
);

// Use the AI's response as context for the next turn
UserMessage userMessage2 = UserMessage.from(
    "Now tell me another one about Python developers."
);

Response<AiMessage> response2 = model.generate(
    systemMessage, userMessage1, response1.content(), userMessage2
);
